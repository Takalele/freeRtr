block pipeline will be opened by this process to the debugger.
once the pipeline opened, the following protocol applies:
  byte: type of packet
n*byte: data payload
the types are the following:

00 - list of registers request...
     data: none

01 - list of registers reply..
     data: n*(byte:size byte:type asciiz:name)
     where type is bitmapped: 80h-additional bitmap available
                              00h-general
                              01h-stack pointer
                              02h-instruction pointer
                              03h-flags

02 - register bitmap request...
     data: dword:number of register (1..max)

03 - register bitmap reply...
     data: n*(byte:begin byte:size asciiz:name)

04 - read register values request...
     data: none

05 - read register values reply...
     data: value of registers...

06 - execute instruction request...
     data: none

07 - execute instruction reply...
     data: dword:result
           where result can be: 00-no error
                                01-invalid opcode
                                02-address out of range
                                03-io accessed
                                04-division by zero

08 - memory read request...
     data: dword:offset
           dword:size

09 - memory read reply....
     data: n*byte:data

10 - memory write request...
     data: dword:offset
           n*byte:data

11 - memory write reply...
     data: none

12 - write register values request...
     data: n*byte:register values...

13 - write register values reply...
     data: none
