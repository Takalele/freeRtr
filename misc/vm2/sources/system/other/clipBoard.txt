block mode pipeline should opened for this process...
once the pipeline has estabilished, the caller sends
command packets, this process replies to them...
in packets, the first dword is the command/status,
the other bytes are data...

the folowing commands are understood:
0 - read bytes from clipboard: data=position,size:dword...
1 - clipboard information: data=maxsize,cursize:dword, name,type:string...
2 - start uploading: data=name,type:string...
3 - upload next part: data=data...
4 - stop uploading: data=none...
