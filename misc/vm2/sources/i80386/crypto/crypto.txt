block mode pipeline should opened for this process...
once opened, commands are sent by the client, and
responses are generated by this process...
the algorithms are numbered in range 0..N-1...

the commands are the following:

 0 - list of ciphers...
     input: dd:cmd;
     output: dd:contextSize; n*( dd:blocksize; asciiz:name )

 1 - list of hashers...
     input: dd:cmd;
     output: dd:contextSize; n*( dd:blocksize; asciiz:name )

 2 - immediate hash data...
     input: dd:cmd; dd:algo; pastring:key,iv; n*db:data
     output: n*db:hashValue

 3 - immediate encrypt data...
     input: dd:cmd; dd:algo; pastring:key,iv; n*db:data
     output: n*db:encryptedData

 4 - immediate decrypt data...
     input: dd:cmd; dd:algo; pastring:key,iv; n*db:data
     output: n*db:decryptedData

 5 - start hash context...
     input: dd:cmd; dd:algo; pastring:key,iv
     output: n*db:context

 6 - add hash data...
     input: dd:cmd; n*db:context; n*db:data
     output: n*db:context

 7 - finish hash context...
     input: dd:cmd; n*db:context
     output: n*db:hashValue

 8 - start cipher context...
     input: dd:cmd; dd:algo; pastring:key,iv
     output: n*db:context

 9 - encrypt cipher data...
     input: dd:cmd; n*db:context; n*db:data
     output: n*db:context; n*db:encryptedData

10 - decrypt cipher data...
     input: dd:cmd; n*db:context; n*db:data
     output: n*db:context; n*db:decryptedData

11 - update cipher context...
     input: dd:cmd; n*db:context; pastring:iv
     output: n*db:context
