this process accepts block mode pipelines from others. once a
pipeline is opened, this process reads commands, processes them,
and sends the reply... the commands are the following:

1 - get card identification....
    input: nothing...
    output: ascii text describing the card...

2 - get mixer names...
    input: nothing...
    output: (min,max:dword; name:asciiz)*num;

3 - get mixer values...
    input: nothing...
    output: (value:dword)*num

4 - set mixer values...
    input: num,value:dword
    output: 1:byte

5 - get buffer info...
    input: nothing...
    output: buf_size:dword

6 - play sample...
    input: (byte)*buf_size
    output: 1:byte

7 - record sample...
    input: nothing...
    output: (byte)*buf_size

8 - play and record sample...
    input: (byte)*buf_size
    output: (byte)*buf_size

9 - dial out a number...
    input: number
    output: byte:status; 1=ok, 0=fail...

10 - hang up call
    input: nothing...
    output: 1:byte


sampling is done at 44000 hz, (left,right:word)...
