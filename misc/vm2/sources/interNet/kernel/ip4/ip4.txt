block pipeline should opened for this process...
once the pipeline has opened, the following commands
must be sent by the caller... this process will answer
to the question, and closes the pipeline...
the only one exception is when data command is issued...
in this case the pipeline changes protocol, and everything
received on it is sent out to the internet, and everything
received from the internet is sent over the pipeline...
'param---'                       'param' 16:localIP
'patred--' dd:seq                'path' dd:seq,max, 16:srcIP,srcMask,trgIP,trgMask,destinationIP
'patadd--' dd:seq, 16:srcIP,srcMask,trgIP,trgMask,destinationIP  'add' dd:result
'patdel--' dd:seq                'del' dd:result
'ifcred--' dd:seq                'iface' dd:seq,max 16:localIP,gateIP,netmask dd:pid bytes:name
'ifcadd--' dd:pid, bytes:name    'add' dd:result
'ifcdel--' dd:seq                'del' dd:result
'getopt--' dd:seq                'option' dd:seq,max,value, bytes:name
'setopt--' dd:seq,value          'set' dd:result
'data----'                       'data'
the layout of data mode packets is the following:
16 bytes: ip6 address...
byte: protocol...
? byets: data...
